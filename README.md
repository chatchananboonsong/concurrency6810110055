# Python Concurrency Laboratory

โปรเจกต์ขนาดเล็กเพื่อศึกษาการทำงานของ Concurrency 3 รูปแบบ โดยแต่ละโปรแกรมทำงานต่อเนื่อง 2 ขั้นตอน

### 1. Threading (I/O Bound)
- **ไฟล์:** `Threading.py`
- **การทำงาน:** เช็คที่ว่าง -> บันทึกการจอง
- **คอนเซปต์:** ใช้สำหรับงานที่ต้องรอ I/O เช่น Database หรือ Network โดยใช้ Memory ร่วมกัน

### 2. asyncio (Asynchronous I/O)
- **ไฟล์:** `Asyncio.py`
- **การทำงาน:** เตรียมไฟล์อีเมล -> กดส่งอีเมล
- **คอนเซปต์:** ทำงานบน Single Thread แต่สลับงานด้วย Event Loop เหมาะกับงานที่ต้องรอพร้อมกันเยอะๆ

### 3. Multiprocessing (CPU Bound)
- **ไฟล์:** `Processing.py`
- **การทำงาน:** ประมวลผลไฟล์ -> สรุปผลรายงาน
- **คอนเซปต์:** ใช้ CPU แยก Core กันทำงาน เหมาะกับการคำนวณหนักๆ เพื่อข้ามขีดจำกัดของ GIL'
